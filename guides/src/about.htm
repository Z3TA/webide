Free JZedit server: webide.se

webide.se is a online web editor/IDE, and cloud shell/workspace for developing and editing web sites/documents, apps and Node.JS services. With integrated dev-tools, cloud hosting service, source control and content management system. Making it easy to both develop and deploy (run, in production) full stack JavaScript web applications from anywhere in any device or platform that comes with a web browser. And Without installing anything, configuration, setup, or touching a terminal or command line!

Why should I use webide.se to edit my web site !?

Besides being an online editor where you can edit the HTML, CSS and JavaScript, It also has a built in static site generator (SSG).
Press F9 or select "Static site generator" from the menu to see the tool-buttons for the SSG. There you can press the preview button, that lets you see changes in real time. Or the WYSIWYG-button which enables edits in the preview window aka. "What you see is what you get".

JZedit

webide.se is a free server for JZedit which is a code editor for JavaScript, CSS, and HTML, with IDE-like features such as auto-complete and built in version control (Mercurial with hggit).

Some features

  • Function list for quick navigation
  • Automatic indentation
  • Auto-completion and function-argument hinting.
  • Connect to FTP/FTPS/SFTP servers and edit files remotely.
  • Google Drive support
  • Chromebook and Smartphone support
  • Grouped file tabs
  • Zoom out Alt+Z with markdown topics highlighted for faster navigation.
  • Static site generator with scripting capabilities using JavaScript, with support for Node.JS modules.
  • WYSIYG and editor side by side - Write HTML without knowing HTML.
  • Source control: Commit from within the editor and show annotations.
  • It's possible to run the editor embedded on a web site.
  • The editor can run on mobile phones and tablets (or any HTML5 compatible device).

For non-technical users

The editor is not only for developers, it can also be setup so that a non-technical user can use it, for example to edit their web site via WYSIWYG, write a blog post, edit documentation, etc. with automated source control management.
For example when using the built in static site generator together with a software repository, it will automatically pull new updates from the repo, merge if necessary, and ask the user to write a commit message before publishing.

Architecture

JZedit is made in JavaScript, HTML and CSS. And it use hardware accelerated HTML 2d Canvas for text rendering.

JZEdit's architecture is plugin and event driven, which makes it fun and easy to extend with new features.

Back-end and front-end is separated in order to make everything work in the browser. The back-end is a Node.JS server. And the client (front-end) can be any browser or nw.js

JZedit has a small core of only a few files. All functionality, even the rending, is implemented via plugins.
Each plugin is a independent unit and do not depend on, or interfere with other plugins.
Plugins can be hot loaded, so while making a plugin, you can reload the plugin without reloading the editor.

What makes JZeidt better then existing editors/IDE's, or what does it do differently !?

You can run both the server and the client on your computer and it will feel just like a normal editor/IDE.
But you can also run the back-end on a server machine, either self hosted, on a public server like webide.se or a server hosted by your organisation - and connect to it from anywhere to get a full developer environment that lets you run Nodejs, and more. Making it possible to develop from a smartphone or Chromebook.

JZedit has a graphical interface for accessing files on remote servers via SSH/FTP. But you can also use the built in terminal, and get a meta experience by ssh:ing to another machine, and run a terminal editor inside the editor.

JZedit is easier to use then some code editors. And if you know HTML/CSS/JavaScript, you will also find it easier to add new features.
JZedit is written in "vanilla" JavaScript, where other similar editors use "compile to Javascript" languages like CoffeeScript and TypeScript.
JZedit has no build step. You do not need to compile! Just refresh it like a normal web page. You can use the editor to make changes to the editor itself, and modules can be hot loaded (hitting F7 reloads only the file/module you are working on).
JZedit does Not use the DOM (content-editable) for text rendering like other web based editors do, JZedit uses 2d Canvas, which makes it faster.

But does it have Vim key-bindings?

Yes, "there is a editor, inside the editor", called Vim mode (vim normal), hit CTRL+SPACE to activate it.

We want hardcore vim users to feel at home, but it's a lot of work, so only expect basic support for now.

Why are you not working on something more important, text-editing is a solved problem

Writing a text editor turned out to be much more fun then anticipated.
And we also think there are room for improvements to make writing code more easier and fun, especially for beginners.

We want to bring professional tools like source control, and static-site-generation to organisations to help them more efficiently manage, share and collaborate with documents and (public or intranet) web sites.

In 1975 an text editor was developed that made use of a video monitor so you could see what you where typing. This editor was however dismissed, because there where really no need to see the text while editing ... So the author ended up calling it the "editor for mortals".
When normal people see "code" they get scared. We want to make "coding" natural for normal people.

The web environment and JavaScript allows rapid development and we hope to experiment with stuff like speech-to-text, and visual aids to show which parts of the code does what, what parts are hot, the scope of variables etc.

Run JZedit locally

Don't want to use the hosted service ? You can install the editor on your own workstation or server!

First install nodejs, (sudo apt install nodejs) or download from Node.js website.
With nodejs you should get a program called npm. Open a terminal/command-prompt and type: npm install -g jzedit.
To run the editor type npm start, or if you installed globally (-g flag) jzedit file-path.
Or if you only want to run the server and open the editor in a browser, type npm run server.
