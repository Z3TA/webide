Using a database

The easiest way to store data persistently is to save it in a file as JSON. Then store it in memory while the app s running. One Gigabyte of data used to be "big data" (eg. alot of data), but now a days servers and even mobile phones have several Gigabytes of memory, which allows you to store all of the apps data in memory for fast access.

When you app grows, it's usually a good strategy to create many specialied services instead of just one gigantic "monolith". Then it will start to become complicated to store the data on the file-system, and you would probably end up implementing a database ...

There are many database systems to choose from and mySQL is currently the most popular.
The cloud IDE lets you create as many mySQL databases as you want.

Is there really any advantages to usinga a SQL database !?

The cloud editor has built in support for mySQL databases.

Creating a database

In the menu, click "Database manager", and then click Create new database.
You then get a dialog box where you type the name of the database.

Running a database query from the Editor

  1. In the menu, click "Database manager".
  2. Select which database to run the query on.
  3. Write the SQL query somewhere, then select it.
  4. Click Run selected text as query in the database manager widget.

The result from the query will show up in a new file.
You can write additional SQL queries in that file, and the result will be appended to the end of the file.

Passwordless autentication (auth_socket)

The editor uses auth_socket mySQL plugin for authentication to the database to make it possible to login securely to the mySQL server without using a password.

The only mysql module I have found that works with auth_socket is mysql2.
And you have to specify authSwitchHandler in the connection options. See example above.
