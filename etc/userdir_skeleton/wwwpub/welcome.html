Static site Generator

To manage larger web sites, the editor has a built in Static Site Generator (SSG).
Show the SSG manager by hitting F9, or click "Static Site Generator" in the menu.

Using the SSG to write a new blog post

  1. In the SSG manager click New Page to create a new page from a template.
  2. Save the page using Ctrl + S. Save the file as:
    /my_web_site/source/blog/name_of_file.html
    Make sure it's saved in the "blog" folder!
  3. Preview the page: Ctrl + F9 or Preview in SSG manager.
  4. Write the blog post and hit Ctrl + S to save when you are done.
  5. Publish the site Ctrl + Alt + Shift + F9 or Publish in SSG manager.

Adding images to a web page

Running Node.JS scripts

Open any JavaScript (Node.JS) file and press F1 (or "Run nodejs script" in the menu) to run it.
Try any of the example files in the /nodejs/ folder.

Unix sockets / named pipes

Use /sock/_somename instead of a port-number. Eg. replace server.listen(80) with server.listen("_nameOfSocket")

All HTTP (and websocket) request starting with http://%USERNAME%.%DOMAIN%/_underscore will be proxied to your /sock/_underscore unix socket.

You might have to add the unix socket as a prefix/path when using modules such as express and socket.io: Example: express().use("/_unixSocket", express.static('public')); or socketio(server, {path: "/_unixSocket/something"});

If you get Error: EADDRINUSE /sock/_yoursock simply delete the file.
(unix sockets created while running a node.js script are normally deleted by the editor when a node.js script stops)

Source control

The editor has built in support for Mercurial "source control management" (SCM), which is similar to Git. The .hggit Mercurial module is installed by default and lets you push to Git repositories!

The built in Static Site Generator (SSG) has a simplified SCM workflow with automatic pull/merge/push to allow mere mortals to use SCM. (The SSG also supports WYSIWYG editing).

Hosting a repository on Github

  1. Go to Github in another browser window/tab to create a new repository (or use an existing one)
  2. Go back to the editor and select "Clone/add Repo" in the menu.
  3. Fill in the http or git url for the repository, and where the files should be saved.
  4. Click Clone repository
