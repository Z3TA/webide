Notes to self for releasing a chrome app


Chrome apps for Chromebooks
===========================

There are two types of Chrome apps:

hosted app: Hosted on a http server. Works like a normal web browser app.
https://developer.chrome.com/webstore/hosted_apps

packaged app: All files are contained in a package. Uses app.background.scripts that runs chrome.app.window.create
Uses API's that are different from web API's. For example localStorage and window.open works very differentingly.

Then there's also "add to shelf" or "add to home screen" A2HS apps that also quacks like a chrome app!
A2HS needs a Service worker and uses manifest.webmanifest 

Also note webapp manifest (manifest.webapp), not to be confused with A2HS, 
that I think browsers used for a short period but is now depreciated and A2HS (manifest.webmanifest) is used instead.
See: https://developer.mozilla.org/en-US/docs/Archive/B2G_OS/Firefox_OS_apps/Building_apps_for_Firefox_OS/Manifest


Test the app
------------
chrome://extensions/

Click "load unpacked extension"

( yes, you also test Apps in chrome://extensions/ )


Start the app
--------------
chrome://apps/

Click on the app icon



manifest.json
=============

name can be max 45 characters
description max 132 characters

manifest_version: 2 (do not touch)
version: Need to be incremented

Hosted app:

"app": {
  "urls": [
    "https://webide.se/"
  ],
  "launch": {
    "web_url": "https://webide.se/"
  }
}

Packaged app:

"permissions": [
		"storage",
		"identity",
		"identity.email",
		"browser"
	],
"app": {
  "background": {
    "scripts": ["chrome-background.js"]
  }
}


version
-------

The version in manifest.json is the version of the manifest.json file and Not the version of the app!
ref: https://developer.chrome.com/webstore/hosted_apps

version: Required. The version of this metadata. Each time you update the metadata this number must be incremented. Up to four dot-separated integers are allowed.
