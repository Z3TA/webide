Free JZedit server: webide.se

webide.se is a online web editor/IDE, and cloud shell/workspace for developing and editing web sites/documents, apps and Node.JS services. With integrated dev-tools, cloud hosting service, source control and content management system. Making it easy to both develop and deploy (run, in production) full stack JavaScript web applications from anywhere in any device or platform that comes with a web browser. And Without installing anything, config, setup, or touching a terminal or command line!

Why should I use webide.se to edit my web site !?

Besides being an online editor where you can edit the HTML, CSS and JavaScript, It also has a built in static site generator (SSG).
Press F9 or select "Static site generator" from the menu to see the tool-buttons for the SSG. There you can press the preview button, that lets you see changes in real time. Or the WYSIWYG-button which enables edits in the preview window aka. "What you see is what you get".

JZedit

webide.se is a free server for JZedit which is a code editor for JavaScript, CSS, and HTML, with IDE-like features such as auto-complete and built in version control (Mercurial with hggit).

While being a "cloud editor" it can also run locally on a PC, or any device that can run Node.JS.

Some features

  • Grouped file tabs
  • Function list for quick navigation
  • Automatic indentation. You never have to indent your code!
  • Smart auto-completion and function-argument hinting using Tab.
  • Connect to FTP/FTPS/SFTP servers and edit files remotely.
  • Zoom out Alt+Z with markdown topics highlighted for faster navigation.
  • Static site generator with scripting capabilities using JavaScript, with support for Node.JS modules.
  • WYSIYG and editor side by side - Write HTML without knowing HTML.
  • Source control: Commit from within the editor and show annotations.
  • It's possible to run the editor embedded on a web site.
  • The editor can run on mobile phones and tablets (or any HTML5 compatible device).

For non-techical users

The editor is not only for developers, it can also be setup so that a non-technical user can use it, for example to edit their web site via WYSIWYG, write a blog post, edit documentation, etc. with automated source control management.
For example when using the built in static site generator together with a software repository, it will pull new updates from the repo, merge if neccesary, and ask the user to write a commit message before publishing.

Architecture

JZedit is made in JavaScript, HTML and CSS. And it use hardware accelerated HTML 2d Canvas for text rendering.

JZEdit's architecture is plugin and event driven, which makes it fun and easy to extend with new features.

Back-end and front-end is separated in order to make everything work in the browser. The back-end is a Node.JS server. And the client (front-end) can be any browser or nw.js

JZedit has a small core of only a few files. All functionality, even the rending, is implemented via plugins.
Each plugin is an independent unit and do not depend on, or interfere with other plugins.
Plugins can be hot loaded, so while making a plugin, you can reload the plugin without reloading the editor.

What makes JZeidt better then existing editors/IDE's, or what does it do differently !?

JZedit is a cloud based editor with separate front-end (browser client) and back-end (Nodejs server).
You can run both the server and the client on your computer and it will feel just like a normal editor/IDE.
But you can also run the back-end on a server machine, either self hosted, on a public server like webide.se or a server hosted by your organisation - and connect to it from anywhere to get a full developer environment that lets you run Nodejs, and more. Making it possible to develop from a smartphone or Chromebook.

JZedit has a graphical interface for accessing files on remote servers via SSH/FTP. But you can also use the built in terminal, and get a meta experience by ssh:ing to another machine, and run a terminal editor inside the editor.

JZedit is easier to use then some code editors. And if you know HTML/CSS/JavaScript, you will also find it easier to add new features to JZedit compared to other editors.
JZedit is written in "vanilla" JavaScript, where other similar editors use "compile to Javascript" languages like CoffeeScript and TypeScript.
JZedit has no build step and you can use the editor to make changes to the editor itself, and hit refresh or hot load to instantly see the new changes in effect.
JZedit does Not use the DOM (content-editable) for text rendering like other web based editors do, JZedit uses 2d Canvas, which makes it faster.
