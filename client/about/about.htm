webide.se

webide.se is JZedit running on a server.

webide.se is a cloud development environment where you can compile static web sites and develop Node.JS micro-services.

webide.se give you web hosting space for static web sites and Node.JS micro services, making it possible to both develop and run (host, in production) full stack JavaScript web applications from any device that have a web browser, without need for plugins or installing anything, or using a terminal or command line!

JZedit

JZedit is a text editor for programming, specialized in JavaScript, CSS, HTML and web development, with IDE-like features such as auto complete and built in version control (Mercurial), and a built in static-site-generator.

JZedit is open source and can run locally on a PC or any device capable of running Node.JS.
It can also run on a headless private server where you connect to it from a browser.

Features

  • Grouped file tabs
  • Function list - For quick navigation, and search of JavaScript functions.
  • Automatic indentation management - Save time not having to indent your code!
  • Zoom out Alt+Z with markdown topics highlighted - Makes it faster to navigate in large files.
  • Smart auto-completion of JavaScript variables Tab.
  • Static site generator with JavaScript support, kinda like classic ASP, but with support for Node.JS modules.
  • WYSIYG and editor side by side - Write HTML without knowing HTML.
  • Connect to FTP/FTPS/SFTP and edit files remotely.
  • Commit from within the editor and show annotations.
  • It's possible to run it embedded on a web site.

Architecture

JZedit's source code is made of vanilla (no transpiling) JavaScript, HTML and CSS, and use HTML Canvas for rendering.

JZEdit's architecture is plugin and event driven, which makes it fun and easy to extend with new features. 

Backend and frontend is separated in order to make everything work in the browser.

JZedit has a small core of only a few files. All functionality, even the rending, are implemented as plugins.
Each plugin is independent - they do not depend on each other - avoiding complexity.
Plugins can be hot loaded, so while making a plugin, you can refresh it without reloading the editor.
